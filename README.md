### `Инструкция`  
```
Интерфейс свагера интуитивно понятен. Все эндпоинты подписаны.
                            
 Для создания факультета с зачисленными студентами, требуется:
 1. Сначала создать список необходимых учеников для зачисления на факультет;
 2. По необходимости работать со списком до создания факультетов;
 3. Создать факультет. Все ученики из ранее созданного списка добавятся 
 в созданный факультет;
 
 
                                  *ВАЖНО!*
  Нельзя создать факультет, а потом уже студентов. Требуется выполнить пункты
  согласно инструкции выше.  
  Студенты в факультеты добавляются исключительно списком, нельзя добавить, удалить
  или редактировать студента, распределенного на факультет. После создания факультета
  список обнуляется и в новый факультет попадают студенты уже из другого созданного списка.
  
  Данное ограничение связано с использованием только одной мапы для учеников. Чтобы 
  студентов  можно было редактировать на любом этапе, необходимо создавать отдельные 
  мапы или множить их в соответствии с логикой появления новых факультетов. На мой взгляд
   это уже работа на уровне курсовой.                            
  Как сделать имею понимание, но не имею времени... 
  
  Согласно д/з необходимые пункты были
  реализованы. Немного отошел от условия  задачи в части создания полей, так как по 
  заданию требовалось создать совершенно одинаковые модели учеников и факультетов. 
  Не увидел смысловой и практической нагрузки в таком действии, поэтому немного отошел 
  от задания и не стал создавать поля ID в этих классах. 
  Реализовал их в сервисах и добавил поле типа Map для студентов
```
